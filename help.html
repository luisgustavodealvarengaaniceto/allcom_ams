Visão Geral do Sistema

Consulta de Status

Consulte o status de equipamentos Allcom JC450 via IMEI, com informações detalhadas de firmware, conectividade e dados técnicos.

Dashboard Analítico

Visualize métricas em tempo real, gráficos de distribuição e análise de status dos equipamentos com relatórios detalhados.

Gestão de Firmware

Compare versões de firmware, defina referências e identifique equipamentos que precisam de atualização.

Exportação Avançada

Exporte dados em formato CSV, Excel ou JSON com formatação profissional e campos organizados.

Especificações do Sistema

Capacidade: Ilimitado (lotes de 100 IMEIs)
Formatos Suportados: .txt, .csv para importação
Exportação: CSV, Excel (.xlsx), JSON
API: JimiCloud FOTA API
Timezone: Conversão automática China → Brasil
Cache: Sistema inteligente com performance otimizada

Guia de Uso Passo a Passo

1

Inserir IMEIs

Digite os IMEIs na caixa de texto (um por linha) ou carregue um arquivo .txt/.csv clicando em "Carregar arquivo .txt".

Exemplo: 869247060051882
868120302622664
868120302622665
2

Validação Automática

O sistema valida automaticamente cada IMEI (deve ter exatamente 15 dígitos numéricos). IMEIs inválidos são destacados em vermelho.

3

Configurar Firmware de Referência

Selecione ou defina um firmware de referência no dropdown para comparação automática. Isso permitirá identificar equipamentos desatualizados.

4

Iniciar Consulta

Clique em "Consultar Status" para iniciar o processamento. Para listas grandes, o sistema divide automaticamente em lotes de 100 IMEIs.

5

Acompanhar Progresso

Observe a barra de progresso detalhada que mostra o status atual: processando, sucesso, erros e progress geral.

6

Analisar Resultados

Visualize os resultados na lista de equipamentos. Use filtros para exibir apenas Online/Offline e busque por IMEI específico.

7

Ver Detalhes e Exportar

Clique em qualquer equipamento para ver detalhes completos. Use os botões de exportação para gerar relatórios em CSV, Excel ou JSON.

Sistema de Gestão de Firmware

Configuração de Firmware de Referência

Defina um firmware de referência para comparação automática com todos os equipamentos consultados.

  1. Selecione um firmware predefinido no dropdown
  2. Ou adicione um firmware customizado
  3. O sistema comparará automaticamente todos os equipamentos
  4. Equipamentos desatualizados serão destacados

Análise de Firmware

Atualizado: Firmware igual ao de referência
Desatualizado: Versão anterior ao de referência
Mais Recente: Versão posterior ao de referência
Desconhecido: Formato não reconhecido

Formato de Firmware Suportado

Padrão: C450Pro_V?[Major].[Minor].[Patch]_[YYYYMMDD ou YYMMDD]

Exemplo 1: C450Pro_2.0.16_20250526
Exemplo 2: C450pro_V2.0.13_250410
Exemplo 3: C450Pro_2.0.08_20241028

Relatórios de Firmware

O sistema gera automaticamente relatórios detalhados mostrando:

  • Distribuição de versões de firmware
  • Percentual de equipamentos atualizados
  • Lista de equipamentos que precisam atualização
  • Prioridades de atualização (Alta/Média/Baixa)
  • Análise comparativa com firmware de referência

Solução de Problemas

Problemas com IMEIs

IMEI Inválido

Sintomas: Mensagem de erro, IMEI destacado em vermelho

Causa: IMEI não tem 15 dígitos ou contém caracteres não numéricos

Solução:

  • Verifique se o IMEI tem exatamente 15 dígitos
  • Remova espaços, traços ou outros caracteres
  • Confirme se não há zeros à esquerda faltando

Arquivo não Carregou

Sintomas: Erro ao carregar arquivo .txt ou .csv

Solução:

  • Verifique se o arquivo está no formato .txt ou .csv
  • Confirme que há um IMEI por linha
  • Remova cabeçalhos ou linhas em branco
  • Verifique a codificação do arquivo (UTF-8 recomendado)

Problemas de Conectividade

Erro de Rede/API

Sintomas: Falha na consulta, timeout, erro HTTP

Solução:

  • Verifique sua conexão com a internet
  • Tente reduzir o número de IMEIs por consulta
  • Aguarde alguns minutos e tente novamente
  • Verifique se não há proxy/firewall bloqueando

Timeout da Requisição

Sintomas: Consulta trava ou demora muito

Solução:

  • Reduza o número de IMEIs (máximo 50 por vez)
  • Verifique a estabilidade da internet
  • Tente em horários de menor tráfego

Problemas com Dados

Dados Incompletos

Sintomas: Informações faltando ou "N/A"

Causa: Normal para equipamentos offline ou com problemas

Solução:

  • Verifique se o equipamento está realmente online
  • Consulte novamente após algumas horas
  • Confirme se o IMEI está correto

Cache Desatualizado

Sintomas: Dados antigos sendo exibidos

Solução:

  • Force uma nova consulta sem usar cache
  • Limpe o cache do navegador (F5 + Ctrl)
  • Use modo incógnito para teste

Problemas de Interface

Layout Quebrado no Mobile

Solução:

  • Atualize o navegador para a versão mais recente
  • Tente rotacionar a tela (landscape/portrait)
  • Zoom out se necessário

Exportação não Funciona

Solução:

  • Verifique se há dados para exportar
  • Permita downloads no navegador
  • Tente um formato diferente (CSV em vez de Excel)

Informações Técnicas da API

Endpoint da API

URL: http://fota-api.jimicloud.com/queryDeviceStatus

Método: POST

Autenticação: Bearer Token

Content-Type: application/json

Configurações de Requisição

Timeout: 30 segundos
Retry: 3 tentativas
Lote Máximo: 100 IMEIs
Rate Limit: Circuit breaker ativo

Formato da Requisição

{
  "imei": [
    "869247060051882",
    "868120302622664",
    "868120302622665"
  ]
}

Estrutura da Resposta

Campos Principais:

  • imei: IMEI do equipamento
  • version: Versão do firmware
  • lastTime: Timestamp da última comunicação
  • iccid: Identificador do SIM card
  • server: Servidor de comunicação
  • log: Log de status do equipamento
  • selfCheckParam: Parâmetros de autodiagnóstico

Processamento de Dados

  1. Conversão de Timezone: China (UTC+8) → Brasil (UTC-3)
  2. Parsing de Firmware: Extração da versão do campo adequado
  3. Cálculo de Status: Online se comunicação < 24h
  4. Análise de selfCheckParam: Extração de dados técnicos
  5. Comparação de Firmware: Com versão de referência
  6. Cache de Resultados: Armazenamento local para performance

Tratamento de Erros

Erro 401: Token inválido ou expirado
Erro 429: Rate limit excedido
Erro 500: Erro interno do servidor
Timeout: Requisição demorou mais que 30s
Network Error: Falha de conectividade
